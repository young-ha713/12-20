MVC : 객체지향프로그래밍에서 MVC란 사용자 인터페이스를 성공적이며 효과적으로 데이터 모형에 관련 시키기 위한 방법론 또는 설계 방식 중 하나이다.  
         MVC패턴은 목적 코드의 재사용에 유용한 것은 물론, 사용자 인터페이스와 응용프로그램 개발에 소요 시간을 줄여주는 형식.  
         디자인 패턴 중 하나. 3가지 역할로 구분한 개발방법론  
  
   Model : 소프트웨어 응용과 그와 관련된 고급 클래스 내의 논리적 데이터 기반 구조를 표현  
                       어플리케이션의 정보, 데이터. 내부 비즈니스 로직을 처리하기 위한 역할  
   View : 사용자 인터페이스 내의 구성요소들을 표현(사용자에게 보여지는 화면)  
                    화면을 만들기 위해 모델로부터 정보를 얻어온다.  
   Controller : Model 과 View를 연결하고 있는 클래스를 대표. Model과 View내의 클래스들 간 정보 교환하는데 사용

설명시 같은 패키지,다른패키지로 구분하여 설명  
public :같은패키지도 접근 가능하고 ,다른패키지도 접근 가능합니다.
private : 같은패키지도 접근 불가능하고 다른패키지도 접근 불가능합니다.오직 클래스 내에서만 접근 가능합니다.  
default:같은 패키지는 접근가능하지만 다른 패키지는 접근 불가능합니다.
*메서드 호출시 매개변수 넘겨주는 타입  
-call by value:매개변수시 값을 넘겨줍니다.그래서 서로 독립된 값을 전달합니다.  
-call by reference:주소값을 넘겨줍니다.그래서 자원을 공유합니다.  
*final:상수입니다.한번 선언하면 값을 변경할 수 없습니다.
static:공용변수입니다.객체를 만들지 않아도 접근이 가능합니다.  
*jvm구조 :code area,stack ,heap이 있습니다.  
코드관리와 final,static은 code area에서 하고  
호출된 메서드와 지역변수는 stack에서 합니다.  
heap은 객체를 관리합니다. 더이상 객체를 참조하지 않을때 가비지컬랙터가 객체를 삭제합니다.  
힙영역과 메서드영역으로 프로그램을 동작시키는곳입니다.  
*싱글톤:객체는 하나만 만들고 필요하다면 주소를 공유하는 기술입니다.

-----------------------------------------------------------------------------------------------------------------  
call by value : 값을 복사하여 처리한다.  
   장점 : 복사하여 처리하기 때문에 안전하다. 원래의 값이 보존 된다.  
   단점 : 복사를 하기 때문에 메모리 사용량이 늘어난다.  
  
call by reference : 값을 인자로 받은 값의 주소로 참조하여 직접 값에 영향을 준다.  
   장점 : 복사하지 않고 직접 참조 하기에 빠르다.  
   단점 : 직접 참조 하기에 원래 값이 영향을 받는다.

private : 같은 클래스 안에서만 쓸 수 있는 접근자.  
        
default : 같은 패키지내에서 사용 가능한 접근자.  
   
protected : 같은 패키지에서 사용 가능, 다른 패키지에서 접근하려면 class의 상속이 필요

의존성 : 어떤 객체가 사용해야 할 객체. new 연산자 등을 써서 만들면 의존성을 자기가 직접 만들어 쓴다 할 수 있다.

스프링프레임워크 특징 : IOC, DI  
  
IOC : Inversion Of Control(제어의 역행, 제어의 역전)  
작성한 메서드나 객체의 호출을 개발자가 결정하는 것이 아닌 외부, 스프링 프레임워크에서 이루어지는 것.  
IOC 장점 : 결합도를 낮추어 유지보수성 향상  
  
DI : Dependency Injection(의존성 주입)  
컴포넌트를 구성하는 인스턴스의 생성과 의존 관계 연결 처리를 IOC 컨테이너에게 위임하는 것  
객체간의 의존성을 자신이 아닌 외부에서 주입하는 개념.  
Setter injection - setter 메서드를 만들고 의존성 주입  
Constructor injection - 클래스 생성자를 만들고 의존성 주입  
Method injection - 일반 메서드를 만들고 의존성 주입  
   
di 사용하는 이유 : 결합도를 낮추기 위해, 응집력을 높이기 위해
